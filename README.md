# MediaVault

MediaVault 是一个 Android 原生媒体库应用，用于管理本机文件夹与 WebDAV / FTP / SMB 远程目录中的视频资源。应用提供媒体入库、搜索、合集、刮削、续播、远程播放缓存和全屏播放器，不依赖 WebView 或 CGI。

| 项目 | 说明 |
|------|------|
| 包名 | `com.mediavault` |
| 最低系统 | Android 8.0（API 26） |
| 编译目标 | Android SDK 34 |
| 当前版本 | **0.6.10**（versionCode 108） |

## 0.5.x 状态与 0.6.x 方向

0.5.x 已基本完成：媒体库可信化、远程可靠性、库维护、备份导出、导入回滚和诊断包已经形成闭环。0.6.x 的主线切换为 **内容发现与观看工作流**，在不拆应用、不引入账号体系的前提下，让用户更快找到想看的内容、继续未看完的内容，并更高效地管理观看队列。

- **一个 APK、一套业务**：不拆独立 Android TV 应用，不引入第二个 `applicationId`。
- **竖屏手机态 + 横屏融合态**：横屏或 Android TV 类设备自动进入融合 UI；竖屏保持手机触控布局。
- **不提供手动“界面”开关**：界面由设备配置和方向自动决定。
- **触控与遥控共存**：融合态保留触控，同时加强焦点链、D-pad 与返回键行为。
- **库维护优先**：新增库诊断、完整问题列表、筛选分页、源健康、远程能力、刮削证据、TMDB 复核队列、备份导出、诊断包、导入回滚和恢复结果可视化，先解决重复、失效、未匹配、缺封面、远程不可解释失败、问题证据留存和迁移恢复等可维护性问题。
- **0.6.x 体验优先**：首页已新增继续观看、最近入库、未看、观看队列、可解释规则推荐、推荐规则筛选、分区数量、筛选数量和当前列表标题提示；搜索页已支持观看状态、来源和排序筛选；详情页已补同合集下一项提示和播放器返回同步；横屏卡片、行列表和播放器控制栏继续强化 D-pad 可达性。账号、多用户、云同步和 Web 管理端继续后置。

## 功能概览

| 区域 | 说明 |
|------|------|
| 媒体库 | 本机目录、WebDAV / FTP / SMB；浏览、搜索、合集、续播进度 |
| 刮削 | 增量扫描；NFO、同目录封面、文件名解析、远程 sidecar / 抽帧 |
| 播放 | 全屏播放、手势 seek、字幕、截图、进度记忆、连播策略 |
| 远程播放 | 网络拉流，记录文件大小、open、Range 和缓存命中状态，已播放片段本地缓存，减少重复下载 |
| 数据管理 | 应用内查看媒体库、刮削记录、封面、远程 prefix/range 缓存占用并清理；导出备份包、导入备份包和生成诊断包 |
| 库维护 | 在刮削侧栏查看库诊断统计、源健康、远程能力、完整问题列表、筛选分页和刮削证据，支持打开详情、单条重刮和从媒体库移除问题项 |
| 0.6.x 工作流 | 首页继续观看、最近入库、首页分区数量、未看入口、观看队列、可解释规则推荐、推荐规则筛选 Chip、首页筛选数量提示、当前列表标题提示、搜索观看状态/来源/排序筛选、详情页下一项提示、播放器返回同步、横屏卡片主色焦点、行列表 D-pad 打开、播放器控制栏默认聚焦和左右循环；后续继续做真机回归 |

## 信息架构

- 底部导航 / 横屏导航轨：**主页、搜索、合集、刮削**。
- **刮削 / 数据 / 目录管理** 只从 **刮削 Tab -> 顶栏设置 -> 右侧侧栏** 进入。
- **库维护** 也在 **刮削 Tab -> 顶栏设置 -> 右侧侧栏**，用于查看重复、失效、未匹配、缺封面等问题。
- **备份与诊断包** 在 **刮削 Tab -> 顶栏设置 -> 右侧侧栏 -> 设置** 中，可导出备份 ZIP、导入备份 ZIP、生成诊断 ZIP。
- 没有底栏“设置”Tab。
- 连播设置在播放器 **列表 -> 连播** 中调整。

## 安装

在 [GitHub Releases](https://github.com/liyw0205/MediaVault/releases) 下载 `MediaVault_<版本>_debug.apk`，安装后按需授予存储和网络权限。

## 使用

### 添加本机媒体

1. 进入 **刮削** Tab。
2. 打开右上角 **设置**，进入 **管理媒体目录**。
3. 点 **添加文件夹**，通过系统文档选择器授权目录。
4. 回到刮削页，对目录执行增量刮削或重扫。

### 添加远程目录

1. 进入 **刮削** Tab -> **设置** -> **管理媒体目录**。
2. 选择 **WebDAV** / **FTP** / **SMB**，填写主机、账号和路径等配置。
3. 使用 **测试连接** 浏览远程目录。
4. 保存后在刮削页扫描远程目录，条目会写入媒体库。

### 播放与续播

- 从主页、搜索、合集或详情页进入播放。
- 播放器支持快进/快退、手势 seek、字幕、截图和连播。
- 详情页会按同合集/同目录分组提示下一项，播放器连播后返回详情页会同步到最后播放的条目。
- 本机与远程条目都会记录播放进度。

## 从源码构建

### 环境要求

- JDK 17
- Android SDK 34
- Gradle Wrapper（仓库内已包含）

### 常规环境

```bash
git clone https://github.com/liyw0205/MediaVault.git
cd MediaVault
export ANDROID_HOME="${ANDROID_HOME:-$HOME/Android/Sdk}"

./gradlew assembleDebug --no-daemon
```

产物位于：`app/build/outputs/apk/debug/app-debug.apk`

### Termux / 本地工作区

Termux 下必须使用仓库脚本，它会临时设置可执行的 aarch64 `aapt2`，并在退出时恢复 `gradle.properties`。

```bash
bash pack_mediavault.sh
# 生成 ../MediaVault_0.6.10_debug.apk
```

不要把 `android.aapt2FromMavenOverride` 长期写入 `gradle.properties`。

## 调试日志

脚本 `log_mediavault.sh` 用于抓取与播放、远程和刮削相关的 logcat。

```bash
./log_mediavault.sh
./log_mediavault.sh 120
LOG_DIR=~/Downloads ./log_mediavault.sh
```

可选环境变量：

- `APP_PACKAGE`：默认 `com.mediavault`
- `LOG_DIR`：日志输出目录

## 应用数据

应用私有数据位于 `/data/user/0/com.mediavault/files/`，包含媒体库、目录配置、远程配置、刮削记录、库诊断快照、封面、远程缓存、播放进度和本地观看队列等。

0.5.x 的库诊断快照写入 `mediavault/library-diagnostics.json`，用于刮削侧栏的库维护统计、源健康、远程能力、完整问题列表和刮削证据。

应用内 **数据** 菜单可以查看占用并清理，无需 adb。

应用内 **备份与诊断包** 菜单可以生成：

- **备份包**：媒体库、目录配置、远程配置、刮削记录、播放进度、观看队列和基础偏好；默认脱敏远程密码和 TMDB Key，不包含封面缓存和远程播放缓存。生成后会回读 ZIP，自检 manifest、schema/kind、必要条目、内容清单和字节数；空库也会写入可导入的空媒体库 JSON。
- **诊断包**：版本、设备、配置摘要、库诊断快照、源健康、远程能力和最近远程错误；生成时不主动触网，源健康使用最近一次“重新检查”的结果。生成后会回读 ZIP，自检 manifest、schema/kind、必要条目、内容清单和字节数。
- **导入备份包**：先预检 ZIP、schema、内容清单、来源版本、媒体/目录/远程数量、刮削记录、观看队列和偏好条目；预检会校验 manifest 内容清单、必要条目、条目字节数、额外条目和可选 JSON/偏好文件，并显示“清单校验”和“可选 JSON”摘要。损坏的播放进度、观看历史、观看队列、字幕偏好或播放 UI 偏好会在确认导入前被拦截；损坏的库诊断快照会被警告并在导入后重新生成。确认导入后会二次解析备份包，二次解析通过才创建内部回滚快照并覆盖媒体库、目录配置、远程配置、刮削记录、播放进度、观看队列和基础偏好；失败会自动回滚。备份中已脱敏的远程密码和 TMDB Key 会尽量保留当前配置，缺失的远程凭据会在导入结果详情中列出，并可直接进入对应远程配置编辑；目录管理会高亮导入后仍需补密码的远程；导入后也可手动复检源健康。
- **导入回滚快照**：可在“备份与诊断包”里查看内部回滚快照数量、时间、大小，手动选择某个快照恢复；恢复前会预览快照 schema、将恢复的文件/偏好数量、媒体库/目录/远程/刮削记录统计和缺失项；也可按需清空，清空不会影响当前媒体库。

## 开发与发布

开发主线遵循以下约束：

- 优先维护既有功能和横屏融合 UI 稳定性。
- 竖屏触控和横屏 D-pad 都必须保持可用。
- 新业务能力默认不加入，除非明确决定。
- 0.6.x 先改善内容发现和观看工作流；首页提供继续观看、最近入库、未看和观看队列等显式入口，推荐先使用同合集、同类型、最近入库、未看等本地可解释规则，并提供本地规则筛选入口，不先做账号、多用户、云同步或独立 TV 应用。
- 每版至少验证冷启动、四 Tab、搜索、刮削侧栏、远程浏览/播放和播放器。
- 每轮开发完成后必须同步更新 README 和本地开发交接文档，写清版本、APK、提交/push 状态和下一步，方便下一次继续开发。

### UI 配色与可读性

当前 UI 是深色工作台风格，基础色在 `app/src/main/res/values/colors.xml`：

| 角色 | 颜色 | 用法 |
|------|------|------|
| 背景 | `mv_bg #0b1220` | 页面底色、播放器遮罩基色 |
| 面板 | `mv_surface #111827` / `mv_surface2 #1f2937` | 列表、抽屉、对话框、卡片面 |
| 主文本 | `mv_text #f9fafb` | 深色背景上的标题和正文 |
| 次文本 | `mv_text_secondary #d1d5db` / `mv_muted #b8c0cc` | 深色背景上的说明和元信息 |
| 弱提示 | `mv_hint #9ca3af` | 仅用于非关键信息，不能当主要正文 |
| 主强调 | `mv_primary #67e8f9` | 主操作、选中态、焦点高亮 |
| 危险 | `mv_danger #fca5a5` | 删除、移除等危险操作文本 |

配色约束：

- 深色背景只配 `mv_text`、`mv_text_secondary`、`mv_muted`、`mv_hint`、`mv_primary`、`mv_danger` 这类浅色文本。
- `mv_on_primary`、`mv_green_on`、`mv_amber_on` 这类深色文本只放在对应的亮色填充按钮、标签或状态块上。
- 新增布局不要直接写硬编码文本色；优先复用 `mv_text`、`mv_text_secondary` 和现有 TextAppearance。
- 禁止浅色背景配浅色字、深色背景配深色字；新增对话框、抽屉、列表项和横屏面板时必须检查可读性。
- 禁用态可以降低透明度，但文本仍应在实际背景上可辨认，不能只靠颜色区分状态。
