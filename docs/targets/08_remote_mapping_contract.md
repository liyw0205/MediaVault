# 目标 08：远程映射变更契约收口

状态：已完成（1.0.15 / versionCode 179）

## 目标

让远程源编辑预览与实际保存行为保持一致，用户能准确知道改了什么、影响哪些记录以及应用不会重写哪些数据。

## 范围

- 区分无映射变化和需要确认的映射变化。
- 列出协议、主机、端口、基础路径中的实际变更字段。
- 按原远程源 id 汇总关联媒体数量和最多三个样本。
- 明确保存策略为更新连接配置后定向复检，不重写库路径。

## 验收

1. 仅账号、密码或名称变化不弹出映射预览。
2. 任一映射字段变化都先展示字段、关联数量和策略。
3. 取消预览不写入配置；确认不重写或删除媒体路径。

## 完成记录

- 预览模型改为 `Unchanged` / `Affected` 明确结果，并提供固定顺序的变更字段。
- 零关联记录的映射变化仍需确认并提示“无关联媒体记录”。
- UI 与完成记录统一采用“复检但不重写路径”策略。
- JVM 单测覆盖非映射字段、全部及单项映射字段、关联过滤、样本上限和零关联情况。
