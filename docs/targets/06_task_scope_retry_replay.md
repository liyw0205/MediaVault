# 目标 06：任务原始范围安全重试

状态：已完成（1.0.13 / versionCode 177）

## 目标

让失败的局部扫描和刮削任务可在不扩大范围的前提下重试，保留原任务的本机目录、远程源和增量/重建意图，并继续以新任务记录保存证据。

## 范围

- 任务记录持久化安全可重放的范围描述，不保存密码或令牌。
- 失败、部分成功和取消的局部扫描/刮削任务可按原范围重试。
- 旧任务或缺失范围任务明确显示不可安全重试。
- 为范围序列化、兼容读取和重放资格增加 JVM 单测。

## 验收

1. 局部重试不会扩大到其他本机目录或远程源。
2. 重试始终创建新任务，不覆盖旧任务记录、错误或统计。
3. 不可安全重试的旧任务没有误导性操作入口。

## 完成记录

- 新任务持久化显式全库/局部范围、本机目录 URI、远程源 id 和增量/重建意图，不保存远程凭据。
- 任务详情仅在范围版本可识别且范围完整时显示重试，并严格沿用原参数创建新任务。
- 旧任务和缺失范围任务继续兼容读取，但明确提示需回到刮削页重新选择范围。
- JVM 单测覆盖范围序列化、旧记录兼容、局部/全库重放资格和非法范围拒绝。
