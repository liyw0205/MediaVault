# 目标 01：详情页安全移出库与来源上下文

状态：已完成（1.0.8 / versionCode 172）

## 目标

补齐单条媒体在详情页的整理闭环。用户可以从详情页确认后仅移除媒体库记录，并可打开该媒体所属来源的维护上下文；任何操作不得删除本机或远程源文件、目录配置或远程配置。

## 范围

- 整理状态区域增加“打开来源”和“移出库”操作。
- 来源上下文按本机目录或远程源预筛选维护中心。
- 移出库前显示媒体标题及明确的数据安全说明。
- 移除成功后刷新库与诊断状态，返回上层页面，避免详情页停留在已不存在的条目上。
- 增加可在 JVM 运行的纯逻辑测试；执行单元测试和 debug APK 构建。

## 完成记录

- 新增 `LibrarySourceContext`，稳定识别远程源和最具体的本机根目录。
- 维护中心支持带远程源或本机根目录的初始筛选；即使该来源当前没有问题，筛选上下文也会保留并显示零问题结果。
- 详情页的移出库操作使用二次确认，只调用应用内部 `LibraryRepository.removeItemByPath`。

## 验收

1. 点击“打开来源”后，维护中心仅显示该本机目录或远程源关联的问题。
2. 点击“移出库”必须二次确认，确认文案明确不会删除源文件、目录或远程配置。
3. 确认移除后仅删除当前媒体库记录，详情页关闭，上层库状态可刷新。
4. 取消确认或条目已不存在时不产生额外写入。

## 非目标

- 不删除用户源文件。
- 不改动本机目录或远程源配置。
- 不在本目标中实现远程 host/basePath 映射迁移。
